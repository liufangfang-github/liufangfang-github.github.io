路由文档
路由API

query

从 v4 开始，删除了 this.props.location.query 对象。删除它的主要原因是允许开发人员创建独特的方式来存储 query 字符串
参考文档
函数组建中获取query使用useLocation() 钩子的实例，钩子不返回解析的query对象，需要解析钩子的实例一获取key/value对参数，
query-string 库可以解析query对象

const queryToStr = require('query-string'); const result = queryToStr.parse(useLocation().search);

params

useParams返回一个包含key/value的对象。useParams()返回的结果类似于是 {id: ‘12’, name: ‘kk’}

基本配置使用

路由这样配置时path: '/user/modify/:id/:name'路由里参数不可以随意添加，要和path里的一一对应

多个参数时的配置和使用

{ path: '/myUrl/:manyParams', element: myElement } 
使用：const params = { id: 12, name: '23'}, navigate(`/myUrl/${JSON.stringify(params)})
获取参数：useParams().manyParams
