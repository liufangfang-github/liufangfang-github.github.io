nuxt生命周期
参考文档

 export default {
  middleware () {}, //服务端
  validate () {}, // 服务端
  asyncData () {}, //服务端
  fetch () {}, // store数据加载
  beforeCreate () {  // 服务端和客户端都会执行},
  created () { // 服务端和客户端都会执行 },
  beforeMount () {}, 
  mounted () {} // 客户端
}

以下报错的原因原因是vue页面的name和component的名字重复了，修改一下compoent的名字或者 vue文件的名字就好

  • created在服务端和客户端都会执行，服务端没有window、localStorage、sessionStorage、cookie，所以created要使用这些对象时，要先判断是否为客户端环境
  • 在created里执行代码又会衍生出另一个问题，如果在created里写了只在客户端执行的代码，这部分代码的效果修改了dom树的话会报错：服务端和客户端dom树不一致。这种情况把代码写到mounted下面就行。
  • nuxt项目是自带vuex的

    nuxt.config.js配置说明，

  • generate: 当运行npm generate会调用该对象中的配置
    • dir: nuxt generate 生成的目录名称。这里可以指定自定义的目录文件名称
    • fallback：设置页面找不到路由时的跳转页面，一般用来设置404页面，在此处设置好后，需要在layouts目录中创建一个名为error.vue(名称不可以更改) 的新布局，设置完这两处后，就可以在项目中测试了，输入不存在的url会自动跳转到这个404页面
  • target
    • static 数据是固定展示，asyncData 不会执行
    • server vue模板中的数据会动态填充，asyncData 会执行
  • components 设置为true时，在项目中可以直接引用～/components下面的组件名称，不需要再次导入
  • plugins 类型Array 数组元素类型String 或 Object
    • 数组类型为object时的属性为src: String (文件的路径)，ssr: Boolean (默认为 true) 如果值为 false，该文件只会在客户端被打包引入。
    • 属性配置的所有插件会在 Nuxt.js 应用初始化之前被加载导入
    • 每次你需要使用 Vue.use() 时，你需要在 plugins/ 目录下创建相应的插件文件，并在 nuxt.config.js 中的 plugins 配置项中配置插件的路径。
